The Enter key test creates a task and navigates to /task/:id.
The task page then makes an SSE call to start the agent stream.
This SSE call goes through the server proxy to the production API.
The production API returns a CloudFront 403 (not authenticated).

The page.url() at this point is the CloudFront error page URL, not localhost.
That's why url.includes("/task/") fails — the browser followed a redirect.

The URL is something like https://d2xsxph8kpxj0f.cloudfront.net/... (403 page).

Fix: Check if the URL at any point included /task/ before the redirect,
or just accept that the task was created (input was consumed).
Better: just check that the URL is NOT the original home page, meaning navigation happened.
