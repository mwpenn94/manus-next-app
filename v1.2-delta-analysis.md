# v1.2 vs v1.1 Delta Analysis

## Key v1.2 Improvements Over v1.1

### 1. Continuous-Run Discipline (MAJOR)
- v1.1: 8 operator-escalation classes that gate-and-wait
- v1.2: Only 3 catastrophic stop conditions (user signal, platform limit, >50% baseline drift)
- **Impact**: Loop never blocks on notifications. All 8 former escalation classes converted to log-and-continue with documented failover.

### 2. Repo Integrity Pre-Check (NEW)
- 5 checks before prompt paste: top-level dirs, docs/, packages/, pnpm-workspace.yaml, config files
- Only critical missing dirs stop the run; everything else logged
- **Impact**: Catches partial-state repos that would produce meaningless output

### 3. Self-Instrumentation Cleanup (NEW)
- Runs after every pass touching candidate's deployed instance
- Deletes harness-created tasks, scheduled tasks, memories, S3 objects, notifications
- Tracks `Cleanup last run` timestamp in STATE_MANIFEST
- **Impact**: Prevents test pollution from corrupting subsequent measurements

### 4. Failover Protocol (REPLACES Operator Escalation)
- Every former escalation class has a fix-now, fix-later, or tag path
- `blocking: true` is ALWAYS false in v1.2
- Compliance veto produces alternative path, never halt
- Convergence proposal written but loop continues
- **Impact**: True autonomous operation

### 5. Convergence as Soft Moment
- v1.1: Convergence = halt, wait for operator
- v1.2: Write proposal, keep running until fresh signal
- **Impact**: Matches continuous-improvement assumption

## Where v1.2 is Better — Applied to Our Implementation

1. **We should adopt the continuous-run discipline**: Our convergence passes should not halt on non-critical issues
2. **We should add cleanup after test runs**: QA testing page should clean up test artifacts
3. **We should implement the failover protocol pattern**: Agent should never apologize/halt, always apply failover
4. **Repo integrity pre-check maps to our health check**: Validate project state before operations
5. **Notifications as post-run reading**: Matches our notification center pattern
