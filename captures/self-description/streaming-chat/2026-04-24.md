### 3. Differentiation Rationale
The streaming chat is the primary interaction surface. Three substantive differentiators from a generic chat implementation:

1. **Multi-tool agentic loop**: The agent executes a turn-based loop with 50+ tool cases (web_search, read_webpage, wide_research, generate_document, browser automation, code execution, etc.), not just text-in/text-out. Each tool call is visualized in the UI with expandable details.
2. **Mode-aware depth control**: Four modes (Standard, Quality, Max, Limitless) control the agent's depth of research, number of tool calls, and whether it auto-continues or asks permission. This is a behavioral differentiator — the same prompt produces different depth of output.
3. **Conversation branching**: Users can fork a conversation at any message to explore alternatives, with branch indicators and the ability to switch between branches. This is a structural differentiator — most chat UIs are linear.

**Ordinary-observer argument:** A reasonable user would perceive this as a research-capable AI assistant that can search the web, read pages, generate documents, and branch conversations — not a simple chatbot.
