# Capability: document-generation

### 1. Status
**Implemented.** Full document generation pipeline with markdown, DOCX, and PDF output, S3 upload, artifact card rendering, and wide-research synthesis.

### 2. Provenance
- Corpus tiers covered: T7 (repo context, Cycle 8 issues)
- Capture dates: 2026-04-24
- Drift window: N/A (first capture)

### 3. Differentiation Rationale
1. **Research-first pipeline**: Documents are generated after wide_research gathers data from multiple sources, not from LLM knowledge alone. This is a quality differentiator.
2. **Multi-format output**: The same content can be output as markdown (inline), DOCX (formatted Word), or PDF (styled). Format selection is driven by user request or system prompt rules.
3. **Artifact card integration**: Generated documents appear as interactive cards in the chat with download buttons, preview thumbnails, and format indicators. This is a UX differentiator.

**Ordinary-observer argument:** A user would perceive this as a research assistant that produces polished documents, not a text generator.

### 4. Input Contract
- Topic/prompt (string)
- Output format (markdown | docx | pdf)
- Research context (optional, from prior wide_research results)
- Title (string)
- Sections (optional structured outline)

### 5. State Machine
```
REQUEST → [WIDE_RESEARCH → SYNTHESIS]? → DOCUMENT_GENERATION → S3_UPLOAD → ARTIFACT_CARD → DONE
                                              ↓
                                         GENERATION_ERROR → RETRY → DOCUMENT_GENERATION
```

**Confidence: 4/5**

### 6. Tool Calls
- wide_research (parallel web searches + read_webpage)
- generate_document (LLM-based content generation)
- Internal: documentGeneration.ts (markdown → DOCX/PDF conversion)
- Internal: storagePut (S3 upload)

### 7. Output Contract
- Document file (markdown/DOCX/PDF)
- S3 URL (public, CDN-backed)
- Artifact card in chat (title, format badge, download link, preview)
- Inline markdown rendering (for markdown format)

### 8. Performance Profile
- Wide research: 10-30s (depends on number of sources)
- Document generation: 5-15s (depends on length)
- PDF rendering: 2-5s
- S3 upload: 1-3s
- Total pipeline: 20-60s typical

**Confidence: 3/5** — Estimates.

### 9. Edge Cases Observed
- User requests PDF but agent generates markdown → addressed in Cycle 8 with format compliance rules
- Very long documents (>10k words) → may hit token limits
- Research returns no results → agent falls back to LLM knowledge
- S3 upload failure → error reported in chat

### 10. Known Weaknesses
- PDF formatting is basic (no complex layouts, tables may not render perfectly)
- DOCX formatting is template-based (limited customization)
- No inline document preview (user must download to view)
- No document versioning (each generation is a new file)
- No collaborative editing

**Confidence: 4/5**
