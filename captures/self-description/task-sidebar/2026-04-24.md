# Capability: task-sidebar

### 1. Status
**Implemented.** Full task sidebar with CRUD, search, filter, status indicators, grouped navigation, mobile bottom nav, and role-based visibility.

### 2. Provenance
- Corpus tiers covered: T7 (repo context)
- Capture dates: 2026-04-24
- Drift window: N/A (first capture)

### 3. Differentiation Rationale
1. **Grouped collapsible navigation**: Sidebar organizes 23+ items into 3 sections (Manus, Tools, More) with auto-expand on active route. This is an IA differentiator — most sidebars are flat lists.
2. **Task status lifecycle**: Tasks have a full lifecycle (active → completed/failed) with visual status badges, progress indicators, and step counts. This is a behavioral differentiator.
3. **Role-based visibility**: Admin-only items are hidden from regular users, with AdminRoute protection on sensitive routes. This is a security differentiator.

**Ordinary-observer argument:** A user would perceive this as a project management sidebar with task tracking, not a generic navigation menu.

### 4. Input Contract
- Task creation: title (string), initial message (string)
- Search: query string
- Filter: status (all | active | completed | failed)
- Navigation: route path

### 5. State Machine
```
SIDEBAR_COLLAPSED ↔ SIDEBAR_EXPANDED
TASK_LIST → TASK_SELECTED → TASK_VIEW
SEARCH_IDLE → SEARCH_ACTIVE → SEARCH_RESULTS
SECTION_COLLAPSED ↔ SECTION_EXPANDED
```

**Confidence: 4/5** — Derived from component analysis.

### 6. Tool Calls
N/A — sidebar is a frontend-only component that calls tRPC procedures (task.list, task.create, task.delete, task.duplicate).

### 7. Output Contract
- Task list rendering (title, date, status badge, step count)
- Navigation state (active route highlighting)
- Search results (filtered task list)
- Mobile bottom nav (5 items: Home, Tasks, New, Projects, More)

### 8. Performance Profile
- Initial render: <100ms
- Task list with 50 items: <200ms
- Search debounce: 300ms
- Navigation transition: ~150ms

**Confidence: 3/5** — Estimates, not measured.

### 9. Edge Cases Observed
- Empty task list → "No tasks yet" empty state
- Long task titles → truncated with ellipsis
- Many tasks (100+) → virtualization not implemented (potential perf issue)
- Sidebar on mobile → collapses to bottom nav
- Admin-only routes for non-admin → "No permission" page

### 10. Known Weaknesses
- No task reordering (drag-and-drop)
- No task folders/categories beyond status filter
- No task sharing between users
- Virtualization missing for very large task lists
- Mobile bottom nav has limited items (5 max)

**Confidence: 4/5**
