## MANUS TASK VIEW — DETAILED OBSERVATIONS

### Task View Structure
- Top bar: "Manus 1.6 Max" (center-left), Collaborate button, Share button, three-dot menu (top-right)
- Chat area: Full width of center panel
- Agent steps shown as collapsible sections with green checkmarks
- "Knowledge recalled(4)" shown as a step
- Each step has a label like "Run check_progress.sh and gather sync status"
- Steps are numbered (e.g., "4 / 4")

### Workspace Panel
- Appears as a small thumbnail at bottom of chat (not a separate panel)
- Shows terminal output preview
- Expandable via "4 / 4" step counter button
- When expanded, shows full workspace with terminal/browser/code tabs

### Task Completion Flow
1. Agent runs through steps (collapsible)
2. Final message from agent (markdown formatted)
3. Green "Task completed" badge
4. "How was this result?" with 5-star rating
5. "Suggested follow-ups" with 3 clickable suggestions
6. Workspace thumbnail at bottom
7. Input area for follow-up messages

### Key Features We're Missing
1. **"Knowledge recalled(N)"** step type — shows when agent uses memory/context
2. **Collapsible step groups** — Manus groups related steps together
3. **5-star task rating** — post-completion feedback
4. **Suggested follow-ups** — AI-generated next steps
5. **"Collaborate" button** — real-time collaboration
6. **Workspace as thumbnail** — compact view that expands (vs our always-visible panel)

### Features We Have That Manus Doesn't (or does differently)
1. **Always-visible workspace panel** — more immediate access to artifacts
2. **Task filter tabs** (All/Running/Done/Error)
3. **Inline search** in sidebar
4. **More nav items** (Skills, Slides, Design, Meetings, etc.)
5. **Category-based suggestion cards** on home
6. **System prompt customization** per-task
